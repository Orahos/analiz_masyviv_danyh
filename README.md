# Аналіз масивів даних: PostgreSQL + Python + Power BI

Цей репозиторій містить матеріали аналітичного проєкту, що об’єднує роботу з базами даних, машинне навчання та інтерактивну візуалізацію даних у Power BI.

## ?? Структура проєкту

- `classicmodel.sql` — SQL-скрипт, який створює базу даних ClassicModels у PostgreSQL (схеми, таблиці, первинне заповнення даними).
- `predictions.py` — Python-скрипт, який підключається до бази, виконує попередню обробку даних, додає нові обчислювані поля, а також виконує кластеризацію клієнтів з використанням алгоритму K-Means.
- `analiz_dannyh_projekt.pbix` — Power BI звіт, який візуалізує дані з бази PostgreSQL та результати кластеризації.

## ?? Як запустити

1. **Створити базу даних PostgreSQL**:
   - Запустіть файл `classicmodel.sql` у середовищі PostgreSQL, щоб створити структуру бази та заповнити її даними.

2. **Обробити та кластеризувати дані в Python**:
   - Запустіть скрипт `predictions.py`. Він під’єднається до бази, обробить дані, додасть додаткові метрики та внесе сегментацію клієнтів у нову таблицю.

3. **Відкрити звіт Power BI**:
   - Відкрийте `analiz_dannyh_projekt.pbix` у Power BI Desktop.
   - Переконайтеся, що з’єднання з базою PostgreSQL налаштоване правильно. Можливо, буде потрібно оновити дані через "Transform Data" > "Data Source Settings".

## ?? Що в результаті

- Інтерактивний звіт Power BI з 7 аналітичними сторінками: від загального огляду до аналізу продажів, клієнтів, співробітників та кластерів.
- Можливість оновлення даних у реальному часі за прямим підключенням до PostgreSQL.
- Повна автоматизація процесу формування звітності — від SQL до візуалізації.

## ?? Дані

Дані є демонстраційними та взяті з модифікованої класичної моделі продажів (`ClassicModels`).
